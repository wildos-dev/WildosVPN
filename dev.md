# План реализации функционала аналогичного Hiddify в WildosVPN

## Анализ текущего состояния проекта

### Текущая архитектура WildosVPN:
- **Backend**: FastAPI + Python + SQLAlchemy
- **Frontend**: React + TypeScript + Chakra UI  
- **VPN Core**: Xray интеграция через gRPC
- **База данных**: SQLAlchemy ORM
- **Аутентификация**: JWT токены

### Существующие компоненты:
- ✅ Административная панель (app/dashboard/)
- ✅ API роутеры (app/routers/)
- ✅ Базовый функционал подписок (app/routers/subscription.py)
- ✅ Шаблоны конфигураций (app/templates/)
- ✅ Xray API интеграция (xray_api/)
- ✅ Модели пользователей (app/models/)

## Ключевые функции Hiddify для реализации

### 1. Умная система определения клиентов
**Функционал**: Автоматическое определение типа VPN клиента по User-Agent
**Поддерживаемые клиенты**:
- Clash/ClashMeta - YAML конфигурации
- SingBox - JSON конфигурации  
- V2Ray/V2RayNG - Base64 ссылки
- Outline - JSON конфигурации
- Shadowrocket, FoXray, Fair - Base64 ссылки
- HiddifyNext, SFA - SingBox JSON

### 2. Пользовательские страницы подписок
**Функционал**: Отдельная веб-страница для каждого пользователя
**URL структура**: `/{XRAY_SUBSCRIPTION_PATH}/{token}/`
**Содержимое**:
- Информация о пользователе (имя, статус, лимиты)
- Использованный трафик и оставшееся время
- Список конфигурационных ссылок
- QR-коды для быстрой настройки
- Кнопки копирования ссылок

### 3. React UI для пользователей
**Функционал**: Отдельное SPA приложение для конечных пользователей
**Особенности**:
- PWA поддержка (Progressive Web App)
- Мультиязычность (i18n)
- Deep linking для мобильных приложений
- Минималистичный дизайн

## Пошаговый план реализации

### Этап 1: Расширение системы определения клиентов
**Задачи:**
1. **Расширить app/routers/subscription.py**
   - Добавить функцию `detect_client_type(user_agent: str)`
   - Реализовать паттерны распознавания для всех популярных клиентов
   - Добавить автоматический выбор формата конфигурации

2. **Обновить client_config словарь**
   - Добавить поддержку всех типов клиентов из Hiddify
   - Настроить правильные MIME-types и форматы

3. **Реализовать get_proper_config() функцию**
   - Логика выбора конфигурации на основе User-Agent
   - Fallback для неизвестных клиентов

### Этап 2: Улучшение пользовательских страниц подписок
**Задачи:**
1. **Обновить HTML шаблон app/templates/subscription/index.html**
   - Добавить современный CSS framework (Tailwind CSS)
   - Улучшить UX/UI дизайн
   - Добавить QR-код генерацию
   - Реализовать копирование в буфер обмена

2. **Добавить JavaScript функциональность**
   - Генерация QR-кодов (qrcode.js)
   - Copy-to-clipboard функции
   - Обнаружение мобильных устройств
   - Deep links для мобильных приложений

3. **Расширить subscription_user_info()**
   - Добавить больше метрик пользователя
   - Форматирование данных для отображения

### Этап 3: Создание React UI для пользователей  
**Задачи:**
1. **Создать новое React приложение app/user-panel/**
   - Настроить Vite сборку
   - Добавить Chakra UI или Tailwind CSS
   - Настроить маршрутизацию

2. **Компоненты пользовательского интерфейса:**
   - `UserInfo.tsx` - информация о пользователе
   - `TrafficStats.tsx` - статистика трафика
   - `ConfigLinks.tsx` - список конфигурационных ссылок
   - `QRCode.tsx` - генерация QR кодов
   - `CopyButton.tsx` - кнопки копирования

3. **Интеграция с backend API**
   - Создать API эндпоинты для пользовательских данных
   - Настроить авторизацию по токенам подписки

### Этап 4: PWA и мобильная поддержка
**Задачи:**
1. **PWA манифест**
   - Создать manifest.json
   - Добавить service worker
   - Настроить иконки и сплеш экраны

2. **Deep linking**
   - Поддержка схем hiddify://import/
   - Интеграция с мобильными приложениями
   - Meta теги для App Store/Google Play

3. **Мобильная оптимизация**
   - Responsive дизайн
   - Touch-friendly интерфейс
   - Оптимизация для медленных соединений

### Этап 5: Мультиязычность и локализация
**Задачи:**
1. **i18n настройка**
   - Интеграция react-i18next
   - Создание файлов переводов
   - Поддержка RTL языков

2. **Языки для поддержки:**
   - Английский (по умолчанию)
   - Русский
   - Фарси
   - Китайский
   - Арабский

### Этап 6: Продвинутые функции конфигураций
**Задачи:**
1. **Расширить app/subscription/share.py**
   - Поддержка всех форматов конфигураций Hiddify
   - Оптимизация конфигураций под клиентов
   - Автоматическое обновление ссылок

2. **Новые эндпоинты API:**
   - `/api/user/{token}/config/clash-meta`
   - `/api/user/{token}/config/singbox`  
   - `/api/user/{token}/config/outline`
   - `/api/user/{token}/config/wireguard`

3. **Система правил маршрутизации**
   - Настраиваемые правила для разных клиентов
   - Оптимизация для различных регионов
   - Балансировка нагрузки

### API эндпоинты для реализации:
```

```
